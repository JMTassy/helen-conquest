## What Was Built

### Phase 1: Memory Infrastructure ✅
- Created `oracle_town/memory/` folder structure with 5 entity types
- Defined `fact_schema.json` for atomic, timestamped governance insights
- Initialized `checkpoints.json` for extraction state tracking
- Created Layer 3 tacit knowledge files (heuristics.md, rules_of_thumb.md)

**Files created:**
- `oracle_town/memory/meta/fact_schema.json`
- `oracle_town/memory/meta/checkpoints.json`
- `oracle_town/memory/tacit/heuristics.md`
- `oracle_town/memory/tacit/rules_of_thumb.md`

### Phase 2: Real-Time Extraction ✅
Built `cycle_observer.py` — extracts governance insights from decision records.

**Key features:**
- Scans decision records and cycle outputs
- Extracts durable facts (decisions, K-violations, emergence signals)
- Appends facts to `entities/{type}/{slug}/items.json` (never deletes)
- Creates daily log entries in `daily/cycle-{NNN}.json`
- Updates checkpoints to avoid reprocessing

**Usage:**
```bash
python3 oracle_town/memory/tools/cycle_observer.py --scan-runs
```

**Output:**
```json
{
  "cycle_num": 1,
  "facts_added": 5,
  "facts_superseded": 0,
  "entities_touched": ["decisions", "invariant_events"]
}
```

### Phase 3: Weekly Synthesis ✅
Built `weekly_synthesizer.py` — maintenance and self-correction.

**Key features:**
- Regenerates `summary.md` for each entity (shows last 12 active facts)
- Detects contradictions (newer facts supersede older ones)
- Updates `heuristics.md` with patterns from the week
- Marks outdated facts as `status: "superseded"` (transparent versioning)

**Usage:**
```bash
python3 oracle_town/memory/tools/weekly_synthesizer.py
```

**Output:**
```json
{
  "timestamp": "2026-01-28T21:09:03",
  "entities_processed": 3,
  "summaries_regenerated": 3,
  "facts_superseded": 0,
  "contradictions_found": 0
}
```

### Phase 4: Memory Lookup API ✅
Built `memory_lookup.py` — advisory interface for governance decisions.

**Key features:**
- `get_heuristics()` — Fetch learned best practices
- `get_rules_of_thumb()` — Fetch practical decision guidance
- `get_entity_summary(type, slug)` — Fetch summary for a specific entity
- `get_decision_history(limit)` — Recent decision outcomes
- `get_invariant_violations()` — All recorded K-violations
- `get_lane_performance()` — Lane effectiveness metrics
- `get_emergence_signals()` — Convergence/divergence detected
- `get_advisory_context()` — Complete context for a mayor decision

**Usage:**
```python
from oracle_town.memory.tools import MemoryLookup

lookup = MemoryLookup()
advisory = lookup.get_advisory_context()

# Use in mayor decision logic (advisory only, never overrides K-Invariants)
```

**Demo:**
```bash
python3 oracle_town/memory/tools/memory_lookup.py --demo
```

## Architecture Overview

### Layer 1: Knowledge Graph
**Location:** `memory/entities/{entity_type}/{entity_slug}/`

Each entity has two files:
- `items.json` — Array of timestamped, categorized facts (append-only, never deleted)
- `summary.md` — Regenerated weekly, shows active facts only

Example fact:
```json
{
  "id": "quorum-abc123",
  "fact": "Quorum-by-class with N=2 succeeded",
  "timestamp": "2026-01-28T21:00:00",
  "status": "active",
  "category": "pattern_success",
  "confidence": 1.0,
  "entity_type": "decisions",
  "entity_slug": "run-001-claim-001",
  "evidence": {"run_id": "RUN_A"},
  "source": "cycle_output"
}
```

**Entity types:**
- `decisions/` — Decision outcomes (SHIP/NO_SHIP)
- `invariant_events/` — K-invariant violations
- `lane_performance/` — Lane effectiveness metrics
- `emergence_signals/` — Convergence/divergence patterns
- `proposal_archetypes/` — Future: proposal type analysis

### Layer 2: Daily Logs
**Location:** `memory/daily/cycle-{NNN}.json`

Raw event transcript:
```json
{
  "cycle_num": 1,
  "timestamp": "2026-01-28T21:00:00",
  "decision": "SHIP",
  "facts_extracted": 5,
  "entities_touched": ["decisions", "invariant_events"]
}
```

### Layer 3: Tacit Knowledge
**Location:** `memory/tacit/`

- **heuristics.md** — Learned patterns (autogenerated weekly)
  - K-Invariant effectiveness
  - Lane performance rankings
  - Proposal archetype success rates
  - Anti-patterns to avoid

- **rules_of_thumb.md** — Practical guidance (human + AI maintained)
  - Quorum strategies
  - Lane selection heuristics
  - Idea incorporation logic
  - Memory safety constraints

## Safety Guarantees

✅ **Guaranteed by design:**

1. **K-Invariants never overridden**
   - Memory is advisory only
   - Mayors see heuristics but decide freely
   - No automatic approval based on memory

2. **Complete auditability**
   - All facts timestamped and categorized
   - No deletions (only supersession)
   - Full version history preserved

3. **Self-correcting**
   - Newer facts supersede contradictions
   - Confidence scores informational only
   - Heuristics fade if contradicted

4. **Fail-safe**
   - If memory missing → governance continues (advisory only)
   - If extraction fails → synthesis still works
   - If synthesis fails → extraction still works

⛔ **Forbidden by architecture:**
- Using memory to approve NO_SHIP unilaterally
- Deleting historical facts
- Assuming memory is complete or correct
- Override K-Invariant checks

## Philosophy

This memory system embodies three principles:

1. **Never Forget** — All facts retained (only superseded, never deleted)
   - Complete history preserved for audit and context

2. **Never Goes Stale** — Weekly synthesis keeps understanding current
   - Outdated facts automatically marked and faded
   - Contradictions detected and resolved

3. **Always Auditable** — Human-readable facts, full version history
   - Transparency: open any JSON and see what the system "knows"
   - Accountability: every fact has timestamp, source, confidence
   - Reversibility: facts can be examined, disputed, corrected
