   File: schemas/hal_output.schema.json
   Purpose: Formal definition of valid HAL outputs

     - mutations + PASS = INVALID (policy escalation guard)

2. HER/HAL VALIDATOR
   File: scripts/her_hal_validate.cjs (executable)
   Purpose: Parse [HER]/[HAL] blocks and validate against schema
   Usage: node scripts/her_hal_validate.cjs <transcript.txt>
   Features:
     - Single-turn validation
     - Multi-turn validation (separated by --- lines)
     - Strict JSON parsing
     - Determinism checks (sorted arrays)

3. HER/HAL MOMENT DETECTOR
   File: scripts/her_hal_moment_detector.cjs (executable)
   Purpose: Detect convergence moments in dialogue logs
   Usage: node scripts/her_hal_moment_detector.cjs <dialogue.ndjson> [K=5]
   Detection Criteria:
     (A) Stabilized compliance (all turns parseable)
     (B) Nontrivial veto + adaptation (BLOCK then PASS/WARN)
     (C) Continuity anchor (refs.ledger_cum_hash present)
   Output: Milestone JSON when pattern detected

4. EXTENDED STREET1 RUNNER
   File: tools/street1_runner_with_dialogue.py
   Purpose: Execute turns and emit BOTH ledger + dialogue logs
   Extends: street1_runner.py with dialogue event emission
   Output:
     - town/ledger.ndjson (same as before)
