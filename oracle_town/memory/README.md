# Oracle Town Self-Observing Memory System

Oracle Town now has a **three-layer, self-maintaining knowledge graph** that learns from its own governance cycles.

This turns Oracle Town from a **static simulator** into a **dynamically learning institution** that can reason about its own decision patterns, detect anti-patterns, and improve over time.

## What This Solves

**Without memory:** Each governance cycle (Day 1, Day 2, Day 3, ...) happens in isolation. The system never learns which proposal types succeed, which lanes are effective, or which K-violations happen most often.

**With memory:** Oracle Town observes itself. After each cycle:
- Real-time extraction captures facts about decisions, invariant violations, and convergence patterns
- Weekly synthesis supersedes outdated understanding and regenerates best practices
- Mayors can reference learned heuristics when making decisions (advisory only, never overriding K-Invariants)

## Three-Layer Architecture

### Layer 1: Knowledge Graph (Long-Term Memory)
**Location:** `memory/entities/{type}/{slug}/`

Structured facts about governance concepts:
```
decisions/
  run-001-claim-001/
    items.json       # Array of facts (never delete, only supersede)
    summary.md       # Regenerated weekly, shows active facts only
  run-002-claim-002/
    items.json
    summary.md

invariant_events/
  blocker-quorum-not-met/
    items.json
    summary.md

lane_performance/
  fast-track-lane/
  democracy-lane/
  ...

emergence_signals/
  convergence-detected/
  high-divergence-warning/
  ...
```

**Key Properties:**
- Facts are timestamped, categorized, and versioned
- Never deleted — outdated facts are marked `status: "superseded"` with a link to the newer fact
- Summaries are cached views, fully regenerable from items.json
- Each fact has a confidence score (informational only)

### Layer 2: Daily Logs (Event Stream)
**Location:** `memory/daily/cycle-{NNN}.json`

Raw cycle transcript:
```json
{
  "cycle_num": 1,
  "timestamp": "2026-01-28T...",
  "decision": "SHIP",
  "facts_extracted": 5,
  "entities_touched": ["decisions", "invariant_events"]
}
```

These are append-only logs of what happened. The real-time extractor mines these for durable facts.

### Layer 3: Tacit Knowledge (Practical Wisdom)
**Location:** `memory/tacit/`

- **heuristics.md** — Learned best practices (autogenerated weekly)
  - K-Invariant patterns that work
  - Lane effectiveness rankings
  - Proposal archetype success rates
  - Anti-patterns to avoid

- **rules_of_thumb.md** — Practical decision guidance
  - Quorum strategies
  - Lane selection heuristics
  - Idea incorporation logic
  - Memory safety constraints

Both files are readable by humans and can be passed to LLMs as advisory context.

## The Compounding Engine

### Real-Time Extraction (Every 30 Minutes)

Script: `tools/cycle_observer.py`

Process:
1. Scan recent decisions and cycle outputs
2. Extract durable facts (multi-cycle relevance)
3. Append to entity items.json
4. Create daily log entry
5. Update checkpoint (avoid reprocessing)

Cost: ~$0.02 per cycle (lightweight)

Example: "Quorum-by-class with N=2 succeeded"
```json
{
  "id": "quorum-abc123",
  "fact": "Quorum-by-class with N=2 succeeded",
  "timestamp": "2026-01-28T21:00:00",
  "status": "active",
  "category": "pattern_success",
  "confidence": 1.0,
  "evidence": {"run_id": "RUN_A"}
}
```

### Weekly Synthesis (Sunday 3:10 AM)

Script: `tools/weekly_synthesizer.py`

Process:
1. Review all facts added this week
2. Detect contradictions (newer facts supersede older ones)
3. Regenerate each entity's summary.md
4. Update heuristics.md with patterns observed
5. Update checkpoint

Cost: ~$0.10 per week (deeper analysis)

Example: After 3 cycles of "fast-track failed", system marks older "fast-track succeeded" as historical and adds a rule "Don't use fast-track for complex changes."

## Integration with Governance

Mayors can access memory to inform decisions (advisory only):

```python
from oracle_town.memory.tools import MemoryLookup

lookup = MemoryLookup()

# Get comprehensive advisory context
advisory = lookup.get_advisory_context()

# Contains:
# - heuristics (learned patterns)
# - rules_of_thumb (practical guidance)
# - recent_decisions (last 5 decision outcomes)
# - invariant_violations (K-violations encountered)
# - lane_performance (which lanes worked well)
# - emergence_signals (convergence/divergence detected)
```

### Safety Guarantees

✅ **Guaranteed by design:**
- Memory cannot override K-Invariants (immutable constitutional rules)
- All decisions are logged (full audit trail)
- Facts are never rewritten, only superseded (transparency)
- Heuristics fade if contradicted (adaptive, not stubborn)
- Advisory-only (mayors decide freely, memory informs)

⛔ **Forbidden by architecture:**
- Using memory to approve NO_SHIP unilaterally
- Deleting historical facts
- Using confidence scores as tiebreakers
- Assuming memory is complete

## How to Use Memory in Practice

### For Mayors (Decision-Making)

```python
# Before making a decision, check what worked before
lookup = MemoryLookup()
context = lookup.get_advisory_context()

# Example: "Should I use fast-track?"
if "fast-track" in heuristics and "failed 3x" in heuristics:
    print("Advisory: Fast-track has high failure rate. Consider standard pathway.")
    # But you can still choose fast-track — it's advisory only
```

### For Supervisors (Input Validation)

```python
# Memory can inform which proposals are likely to converge
lane_perf = lookup.get_lane_performance()
if len(lane_perf["democracy_lane"]) > 5:
    print("Advisory: Democracy lane very active. Expect convergence on inclusive proposals.")
```

### For Synthesizers (Memory Maintenance)

```python
# Synthesis runs weekly, automatically:
# 1. Reads all facts added this week
# 2. Detects contradictions
# 3. Supersedes outdated understanding
# 4. Regenerates summaries
# 5. Updates heuristics.md

# No manual work needed—just let it run via cron
```

## Philosophy

This memory system embodies three principles:

1. **Never Forget** — All facts retained (only superseded, never deleted)
2. **Never Goes Stale** — Weekly synthesis keeps understanding current
3. **Always Auditable** — Human-readable facts, full version history

The design is inspired by Moltbot's three-layer memory architecture, adapted for Oracle Town's governance context.
