🔐 SAFETY GUARANTEES

✓ K15: No Receipt = No Execution
  - Kernel must approve every action
  - Fails closed if unreachable
  - Can't bypass with code changes

✓ Immutable Audit Trail
  - Every decision logged to kernel/ledger.json
  - Cannot be modified retroactively
  - Full history of all actions

✓ Authority Separation
  - Skill layer (this code) proposes actions
  - Kernel layer (separate process) decides
  - Ledger layer (separate file) records
  - Three independent systems = three checks

✓ Determinism (K5)
  - Same input → identical decision every time
  - Reproducible and auditable

📊 WHAT'S HAPPENING UNDER THE HOOD

4-Step Flow (Per Action):

  1. Skill prepares action (fetch URL, collect evidence)
     └─ [Local: no kernel involved yet]

  2. Skill requests kernel approval
     └─ [Request via socket: KernelClient → kernel daemon]

  3. Kernel decides (ACCEPT or REJECT)
     └─ [Response via socket: kernel daemon → KernelClient]

  4. Skill executes (only if approved)
     └─ [Action taken: fetch URL, execute command, write memory]

All steps logged to kernel/ledger.json (immutable)

🔍 MONITORING

Watch ledger in real-time:
  tail -f kernel/ledger.json | jq '.'

Count decisions:
  cat kernel/ledger.json | jq '.entries[] | .status' | sort | uniq -c

Find all fetch decisions:
  cat kernel/ledger.json | jq '.entries[] | select(.action_type=="fetch")'

Find all rejections:
  cat kernel/ledger.json | jq '.entries[] | select(.status=="REJECT")'

      def __init__(self):
          super().__init__("YourSkill")
      
      def your_action(self, param):
          decision = self.request_approval(
              action_type="your_action",
              target=param,
              evidence={...}
          )
          self.verify_approval(decision, "your_action", param)
          return do_action(param)
