THE 3-SKILL SYSTEM
═══════════════════════════════════════════════════════════════════════════════

You have 3 kernel-approved skills:

1. FetchSkill
   ───────────
   What: Fetch URLs with approval
   Instead of: requests.get(url)
   Use: fetch = FetchSkill(); fetch.fetch(url)
   Kernel decides: Yes/No based on policy

2. ShellSkill
   ──────────
   What: Execute commands with approval
   Instead of: subprocess.run(cmd)
   Use: shell = ShellSkill(); shell.execute(cmd)
   Kernel decides: Yes/No based on policy

3. MemorySkill
   ───────────
   What: Store/retrieve data with approval
   Instead of: self.data[key] = value
   Use: memory = MemorySkill(); memory.write(key, value)
   Kernel decides: Yes/No for writes (reads are free)

════════════════════════════════════════════════════════════════════════════════

WHAT HAPPENS WHEN YOU USE THEM
══════════════════════════════════════════════════════════════════════════════

When you call a kernel skill:

1. Skill prepares request
   ├─ Gather evidence (URL, command, data)
   └─ Send to kernel daemon

2. Kernel decides
   ├─ Check policy
   ├─ Approve or reject
   └─ Log decision to kernel/ledger.json

3. Skill executes (if approved)
   ├─ If approved: action happens, result returned
   └─ If denied: RuntimeError raised, agent handles it

4. Complete audit trail
   └─ kernel/ledger.json has every decision with timestamp

KEY GUARANTEES
═════════════════════════════════════════════════════════════════════════════

✓ K15: No receipt = no execution
  • Agent CANNOT execute without kernel approval
  • Fails closed (denies if kernel unreachable)
  • No backdoors or escape routes

✓ Immutable ledger
  • Every decision logged
  • Cannot be modified after logging
  • Compliance-ready

✓ Determinism
  • Same input → same decision
  • Fully reproducible
  • Tests are reliable

✓ Explicit failures
  • Every error includes reason
  • No silent failures
  • Debugging is straightforward

TROUBLESHOOTING
═════════════════════════════════════════════════════════════════════════════

Problem: Agent says "Kernel daemon unreachable"
Solution: Start kernel daemon (python3 oracle_town/kernel/kernel_daemon.py &)

Problem: Agent says "Kernel rejected action"
Solution: Check ledger (cat kernel/ledger.json | jq '.entries[-1]')
          Update policy or fix the URL/command

Problem: No ledger entries appearing
Solution: Make sure kernel daemon is running
          Check kernel_daemon.log for errors

Problem: Want to debug what kernel is doing
Solution: 1. Run automatic mode: python3 openclaw_automatic_mode.py
          2. Check automatic_mode_results.json
