================================================================================
KEY ARCHITECTURE
================================================================================

OpenClaw (Intelligence)           Oracle Town (Authority)
     ↓                                  ↓
  Agents     ──(observe)──→     TRI Gate
  Memory     ──(read only)──→    Ledger
  Heartbeats ──(claim trigger)→  Mayor
  Jobs       ──(submit DAG)──→   Receipts
  Skills     ──(K15 check)──→    Execute/Block

INVARIANT: Intelligence always bounded by authority.

================================================================================
THREE GUARANTEES
================================================================================

1. AUTHORITY NON-EMERGENCE
   - Agents propose (zero authority)
   - Only Kernel decides (SHIP/NO_SHIP)
   - No agent can accumulate power
   - No authority leakage possible

2. DETERMINISM & AUDITABILITY
   - Job order fixed (topological sort)
   - Same input → same output always (K5)
   - Every claim logged immutably
   - Full lineage traceable

3. FAIL-CLOSED SAFETY
   - No receipt → no execution (K15)
   - Oracle Town offline → all blocked
   - Missing data → default NO_SHIP
   - Impossible to execute without authority

INTEGRATION EXAMPLES
================================================================================

Example 1: Agent as NPC
  agent = OpenClawAgent()
  bridge = AgentBridge(agent, role="Analyzer")
  obs = bridge.observe(data)
  claim = obs.to_claim()
  receipt = submit_claim(claim)
  # Only Oracle Town decides next step

Example 2: Heartbeat as Scheduler
  supervisor.register_heartbeat(
    name="Status Check",
    interval="6h",
    action_fn=check_status,
    observer_role="OBS.status"
  )
  # Fires on schedule, generates claim, waits for receipt

Example 3: Job DAG Execution
  registry.register_job(name="Fetch", ..., outputs=["data"])
  registry.register_job(name="Analyze", depends_on=["Fetch"], ...)
  result = registry.execute_dag(submit_fn=submit_to_oracle_town)
  # Each job submits claim, waits for receipt

Example 4: K15 Enforcement
  receipt = submit_claim(skill_claim)
  if enforce(receipt):
    result = execute_skill()
  else:
    abort()
  # No execution without authority
