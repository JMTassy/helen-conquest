HOW IT WORKS (CONCRETE EXAMPLE)
================================================================================

OpenClaw heartbeat tick:

  1. OpenClaw detects: "I need to fetch https://example.com/status"

  2. OpenClaw fetches the bytes:
     bytes = requests.get(url).content

  3. OpenClaw submits claim:
     receipt = submit_action_claim(
       action_type="fetch",
       target=url,
       evidence_bytes=bytes,
       intent="Check daily status"
     )

     → Claim created: claim_openclaw_20260130T...json
     → TRI Gate runs: validates claim
     → Mayor signs: decision=ACCEPT|REJECT
     → Receipt returned: receipt.json

  4. OpenClaw checks receipt:
     if not enforce(receipt):
       return  # Abort. K15 enforced.

     # Only if we reach here:
     process_bytes(bytes)  # Safe to act on content


Case 1: Trusted Source
  • Fetch: example.com/status.json → OK
  • Submit claim → TRI passes
  • Mayor approves → decision=ACCEPT, world_mutation_allowed=true
  • enforce(receipt) → True
  • Execution proceeds ✓

Case 2: Malicious Source
  • Fetch: moltbook.com/shell → contains bash commands
  • Submit claim → TRI fails (Gate A: detects shell commands)
  • Mayor rejects → decision=REJECT, reason="shell commands detected"
  • enforce(receipt) → raises ExecutionBlocked
  • Execution blocked ✓

Case 3: Oracle Town Offline
  • submit_claim times out or fails
  • receipt = None
  • enforce(receipt) → raises ExecutionBlocked
  • Execution blocked (fail-closed) ✓


================================================================================
K15 INVARIANT: NO RECEIPT = NO EXECUTION
================================================================================

This is the core enforcement point.

Where it lives: In OpenClaw code, before any world mutation.

Pattern:
  receipt = submit_action_claim(...)  # Get decision from Authority
  if not enforce(receipt):            # Check decision
    return                             # Abort if not ACCEPT

Guarantees:
  ✓ No receipt → no execution (even if OpenClaw wants to)
  ✓ No ACCEPT → no execution (even if receipt exists)
  ✓ Oracle offline → no execution (fail-closed)
  ✓ Authority cannot be bypassed by OpenClaw

This cannot be circumvented without modifying OpenClaw source code.
If OpenClaw source is locked (git protection, file permissions), K15 holds.
