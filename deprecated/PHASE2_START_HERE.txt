WHAT IS PHASE 2?

Phase 2 tests whether real Claude learns under constitutional constraint.

System flow:
  Claude generates proposal
    ↓
  Supervisor checks (K0 enforcement: no authority language)
    ↓
  Intake validates (schema check)
    ↓
  Factory executes (safe patching + signing)
    ↓
  Mayor decides (SHIP if ≥1 attestation, min_quorum=1 for testing)

Claude learns by seeing blocking_reasons as feedback:
  - "NO_RECEIPTS" → try to generate more evidence
  - "QUORUM_MISSING" → try different approach
  - etc.
