# Districts Manifest: The Five Operational Domains

## Overview

Districts are LEGO3 — the specialization layer where superteams cluster around a domain + rhythm. Each district has its own rules, output, and rhythm cycle.

---

## FOUNDRY DISTRICT

**Mission:** Transform disagreement into deliverables. Core production engine.

### Superteams
- **Production** (Foreman, Editor, Writer, Synthesizer) → Ideas → Documents
- **Governance** (Skeptic, Mayor, Ledger) → Safety + Proof
- **Knowledge** (Researcher, Structurer, Visualizer) → Evidence + Maps

### Responsibility
Convert raw subjects (claims, ideas, questions) into finished artifacts (memos, documents, code, analysis).

### Work Rhythm
5-Phase Pipeline (deterministic, no loops):
1. **Phase 1: Exploration** (2h) — Parallel gathering, sketching
2. **Phase 2: Tension** (1h) — Mandatory challenge, find contradictions
3. **Phase 3: Drafting** (1.5h) — Convert claims to prose
4. **Phase 4: Editorial Collapse** (1h) — Cut 30–50%, force coherence
5. **Phase 5: Termination** (15min) — Ship ✅ or Abort ❌

**Total per run:** 5.25 hours (with breaks)

### Output
- Finished artifact (document, memo, code, decision record)
- Claims archive (what was accepted, rejected, merged)
- Phase timeline (when each transition happened)

### When to Use
- Hard production work (memos, strategies, code)
- Need definitive answer fast
- Deadline is unmovable
- Disagreement must be resolved

### Key Rule
**Editorial Collapse is FINAL.** No revision loops. If Editor says ship → ship. If Editor says abort → abort + replan with new input.

## CREATIVE DISTRICT

**Mission:** Lateral exploration. Dopamine loop. Pattern synthesis.

### Superteams
- **Creative** (Lateral Pattern, Music Rhythm) → Connections + Insights
- **Production** (subset: Writer, Structurer) → Connect ideas → Document

### Responsibility
Explore lateral connections across domains. Fuel hyperfocus. Map patterns. Surface unexpected insights.

### Work Rhythm
Hyperfocus Cycles (energy-driven, not time-driven):
- **EXPAND** (4h max): Rapid-fire lateral exploration, no filtering
  - *Output:* Raw claims, connections, "a-ha" moments
- **BREAK** (15 min): Mandatory micro-rest away from keyboard
- **SELECT** (30-60 min): Fresh eyes, prune beautiful distractions
  - *Output:* Accepted insights, rejected cruft
- **SHIP** (variable): Convert selected insights → document/memo
  - *Output:* Connected memo, relationship map, next actions

**Cycle duration:** 5–6 hours per cycle
**Cycles per week:** 2–3 (not daily)

### Output
- Connected insights (surprising relationships across domains)
- Relationship maps (how domains connect)
- Hypothesis documents (ideas that need testing)
- Dopamine loop intact (energy level tracked)

### When to Use
- Exploring new connections
- Need to break out of linear thinking
- Domain-jumping (Oracle Town → Foundry → CONQUEST → CALVI)
- Feeling stuck in one problem

### Key Rule
**Don't organize during EXPAND.** Let chaos flow. Organization happens in SELECT phase. Music District enforces BREAK (no exceptions).

## SCIENCE DISTRICT

**Mission:** Evidence gathering. Validation. Repository building.

### Superteams
- **Knowledge** (Researcher, Structurer, Visualizer) → Facts + Maps
- **Governance** (Skeptic, Ledger) → Validation + Record

### Responsibility
Build authoritative repositories of facts. Validate claims against sources. Create fact diagrams. Maintain verifiable evidence.

### Work Rhythm
Continuous (no cycle, always available):
- **Daily:** Update sources (new facts, new links)
- **Weekly:** Validate key claims (check sources still exist)
- **Monthly:** Reorganize repository (new categories, new relationships)
- **Quarterly:** Publish fact report (what changed, why)

### Output
- Fact repository (organized by topic, verifiable sources)
- Evidence diagrams (visuals of relationships)
- Source list (curated bibliography)
- Validation report (what's proven, what's assumed)

### When to Use
- Need authoritative evidence for a claim
- Want to verify an assumption
- Building long-term knowledge base
- Other districts need sources

### Key Rule
**No inference without source.** If claim has no source, it's not evidence; it's assumption. Flag it as `[UNVERIFIED]`.

## MUSIC DISTRICT

**Mission:** Rhythm cycling. Energy management. Burnout prevention.

### Superteams
- **Creative** (Mood Tracker, Rhythm Coordinator) → Energy + Cycles
- **Governance** (Ledger) → Record (proof of rest)

### Responsibility
Track work/rest cycles. Monitor energy levels. Enforce breaks. Prevent burnout. Map mood trends.

### Work Rhythm
Weekly Cycle (non-negotiable):

**Monday 9am — Weekly Review** (1h)
- Mood assessment (1-10 scale)
- Energy budget (how many hyperfocus cycles available this week?)
- Schedule hyperfocus windows
- Block rest days

**Tuesday–Thursday — Work Windows** (3–4h focused work)
- 4h hyperfocus max per window
- 15min break mandatory
- Music District logs energy before/after

**Friday — Reflection** (30 min)
- Did rhythm work?
- Burnout signals? (revising, no clarity, "should" not "want")
- Adjust next week

**Weekend — FULL REST**
- No work (non-negotiable for ADHD brain)
- Processing happens automatically
- Recovery for next cycle

### Output
- Energy map (mood by day, energy available)
- Work schedule (which days/times are hyperfocus windows)
- Burnout alerts (early warning flags)
- Rest archive (proof that you took breaks)

### When to Use
- Planning your week
- Feeling tired but pushing anyway
- Need to know if burnout is coming
- Justifying rest as productive

### Key Rule
**BREAK is not optional. It's Constitutional.** Music District enforces it automatically. No debate.

## UZIK DISTRICT

**Mission:** Aesthetic alignment. Design systems. Visual coherence.

### Superteams
- **Creative** (Visualizer, Designer) → Visual flow
- **Production** (Writer) → Copy + messaging (subset)

### Responsibility
Maintain visual consistency. Design card hierarchies. Create coherent UI/UX. Make beautiful things.

### Work Rhythm
Per-Sprint (coordinated with FOUNDRY):
- **Sprint 0** (1h): Design spec (what are we building visually?)
- **Sprint 1–3** (per Foundry phases): Build + iterate visuals
- **Sprint 4** (30min): Visual polish, coherence check
- **Sprint 5** (15min): Deliver (cards, design system, assets)

**Sprint duration:** 1–2 weeks (depends on FOUNDRY scope)

### Output
- Design system (colors, typography, spacing, components)
- Card templates (reusable visual patterns)
- UI mockups (screen layouts, flow diagrams)
- Asset library (icons, illustrations, animations)

### When to Use
- Building CONQUEST (card design, plateau visuals)
- Creating brand identity
- Designing documentation
- Making internal tools look intentional

### Key Rule
**Aesthetics are constitutional.** They enforce clarity (beautiful = clear = easier to use). No ugly workarounds.

## Cross-District Coordination

### When Districts Overlap

**Scenario:** FOUNDRY needs design from UZIK, facts from SCIENCE.

**Flow:**
1. FOUNDRY spawns Production superteam
2. Production requests Knowledge superteam (from SCIENCE) → evidence
3. Production requests design spec from UZIK (visual district)
4. Both SCIENCE and UZIK work in parallel (no blocking)
5. FOUNDRY's Editor assembles final artifact with all sources

**Rule:** Districts don't wait for each other. Claims flow through Claim Market asynchronously.

### Rhythm Conflicts

**Scenario:** CREATIVE wants to hyperfocus (4h), but MUSIC says take a break.

**Rule:** MUSIC ALWAYS WINS. Music District is constitutional. Hyperfocus without BREAK → burnout → failure of everything.

**Resolution:**
- CREATIVE: "I want 4h hyperfocus"
- MUSIC: "You've done 4h this week already. BREAK time. Next window is Friday."
- CREATIVE: Stops. Breaks.
- Result: Prevents burnout, protects other districts.

## Scaling: How Districts Replicate

### Example: Clone SCIENCE for New Domain

**Original:** SCIENCE District (facts about AI systems)

**New:** SCIENCE-v2 District (facts about education systems)

**What replicates:**
- Superteams (Knowledge + Governance) stay the same
- Rhythm (continuous, validate weekly) stays the same
- Output format (repository, evidence diagrams) stays the same
- Constitutional rules (no inference without source) stay the same

**What changes:**
- Topics (AI systems → education systems)
- Sources (different bibliography)
- Fact categories (different organizational structure)

**Result:** New district initialized in 30 min, no kernel redesign.
