## 🎯 Five Operating Modes

Once built, you can run ORACLE TOWN in five ways:

### Mode 1: Daily Digest 📅
```bash
python3 oracle_town/os_runner.py --mode daily --time 09:00
```
→ Runs every 24 hours
→ Collects observations, generates insights
→ Your town learns automatically

### Mode 2: Continuous Monitoring ⚡
```bash
python3 oracle_town/os_runner.py --mode continuous
```
→ Checks every 30 minutes
→ Real-time alerts for urgent decisions
→ Continuous learning

### Mode 3: Weekly Synthesis 📊
```bash
python3 oracle_town/os_runner.py --mode weekly --day friday
```
→ Deep pattern detection every Friday
→ Recommends policy improvements
→ Strategic insights

### Mode 4: Interactive Explorer 🤖
```bash
python3 oracle_town/interactive_explorer.py
```
→ Ask questions in natural language
→ Town searches memory and answers
→ Explore past decisions

### Mode 5: Scenario Simulator 🔮
```bash
python3 oracle_town/scenario_simulator.py --scenario policy.json
```
→ Test policy changes before applying
→ Impact analysis on past decisions
→ Make informed choices

## 🎓 Key Concepts

**Self-Evolution**
- Town measures its own accuracy
- Adjusts thresholds based on performance
- Policies version for audit trail
- Improves continuously

**Memory & Precedent**
- Searches past decisions for similar claims
- Uses historical accuracy to inform decisions
- "Similar vendors succeeded 100% of the time"

**Insight Engine**
- Detects patterns in claims
- Finds anomalies
- Spots emerging themes
- Measures district accuracy
- Identifies opportunities

**Policy Versioning**
- New policy version when improvements made
- Old decisions pinned to version current when made
- Audit trail maintained
- Evolution enabled without breaking history
