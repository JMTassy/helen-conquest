## 📁 Canonical Artifacts (JSON-First)

### 1. Briefcase (Kernel Input)
**Location:** Generated by Concierge
**Schema:** `oracle_town/schemas/briefcase.schema.json`
**Purpose:** Single source of truth for what must be verified

**Fields:**
```json
{
  "run_id": "RUN_...",
  "claim_id": "CLM_...",
  "claim_type": "COMMENTARY | CHANGE_REQUEST",
  "required_obligations": [...],
  "requested_tests": [...],
  "kill_switch_policies": [...]
}
```

**Constitutional Rule:** `additionalProperties: false` - no confidence allowed

---

### 2. Attestation (Truth Primitive)
**Location:** `attestations_ledger.jsonl` (append-only)
**Schema:** `oracle_town/schemas/attestation.schema.json`
**Purpose:** Proof that an obligation was satisfied

**Fields:**
```json
{
  "run_id": "RUN_...",
  "claim_id": "CLM_...",
  "obligation_name": "unit_tests_green",
  "attestor": "CI_RUNNER",
  "policy_match": 1,  // Binary: 0 or 1 ONLY
  "payload_hash": "sha256...",
  "timestamp": "2026-01-21T..."
}
```

**Constitutional Rule:** `policy_match ∈ {0, 1}` - NO soft values

---

### 3. DecisionRecord (Mayor Output)
**Location:** `decisions/decision_{run_id}.json`
**Schema:** `oracle_town/schemas/decision_record.schema.json`
**Purpose:** Binary shipping verdict with explicit blockers

**Fields:**
```json
{
  "run_id": "RUN_...",
  "claim_id": "CLM_...",
  "decision": "SHIP | NO_SHIP",
  "kill_switch_triggered": false,

  "code_version": "oracle_town_v2.0.0"
}
```

**Constitutional Rule:** Only Mayor may emit this file

### 4. BuilderPacket (District Output)
**Location:** Generated by district concierges
**Schema:** `oracle_town/schemas/builder_packet.schema.json`

## 🎯 Mayor V2 Decision Semantics (Mathematical)

### Satisfaction Predicate
```
Let O_HARD = {o ∈ O : o.severity = "HARD"}
Let A = attestations for this run

sat(o, A) ≡ ∃a ∈ A :
  (a.obligation_name = o.name) ∧
  (a.policy_match = 1)
```

### Constitutional Rules (Applied in Order)
```
DECIDE(Briefcase, Attestations) =
  IF kill_switch_triggered → NO_SHIP
  ELSE IF ∃o ∈ O_HARD : ¬sat(o, A) → NO_SHIP
  ELSE → SHIP
```

### Properties
- ✅ **Deterministic:** Same inputs → same output
- ✅ **Monotonic:** Adding satisfied attestations can only change NO_SHIP → SHIP
- ✅ **Transparent:** `blocking_obligations` explicitly lists missing receipts
- ✅ **No Reasoning:** Pure lookup, no inference, no confidence

## 🔒 Confidence Boundary Enforcement

### Where Confidence is ALLOWED (Cognition Layer)
✅ Street agent reports (ChatDev simulation)
✅ Building aggregations (QI-INT scoring)
✅ District narratives (`BuilderPacket.confidence`)
✅ UI visualizations (town simulation)
✅ Telemetry files (`telemetry.jsonl`)

### Where Confidence is FORBIDDEN (Kernel)
❌ `Briefcase` schema (no confidence field)
❌ `Attestation` schema (no confidence field)
❌ `DecisionRecord` schema (no confidence field)
❌ Mayor V2 logic (only reads attestations)
❌ Factory logic (only executes tests)

### Enforcement Mechanism
**Schema Validation:**
```json
{
  "briefcase.schema.json": {
    "additionalProperties": false,  // Fail closed
    "required": ["run_id", "claim_id", "required_obligations"]
    // NO "confidence", "score", or "recommendation"
  }
}
```

**EV District Obligation:**
- `briefcase_schema_valid` ensures no telemetry leaks into kernel

## 🎓 Key Insights

### 1. Simulation UX with Governance Kernel Inside
**Cognition layer** (Streets, Buildings, Districts) can use confidence, scores, narrative → makes system feel alive and user-friendly

**Governance kernel** (Factory, Mayor) uses pure truth checking, no reasoning → makes system trustworthy and deterministic

**Result:**
- ✅ Immediate demo value (visual agents, town metaphor)
- ✅ Constitutional rigor (kernel enforces NO RECEIPT = NO SHIP)
- ✅ Migration path (swap mocks → real tests without architecture change)

---

### 2. The Kernel Boundary is Data, Not Code
The separation between Cognition and Truth is enforced by **schemas**, not by code organization:

```
BuilderPacket.confidence = 0.95  ✅ ALLOWED (has confidence field)
                ↓
         Concierge filters
                ↓
Briefcase.confidence = ???       ❌ REJECTED (no such field in schema)
```

This is **fail-closed by design** - unknown fields cannot leak across the boundary.

---

### 3. Districts Scale Horizontally, Not Vertically
Adding a new district:
```python
# CORRECT: Add new obligation emitter
class BusinessConcierge:
    def analyze_claim(self, claim, metadata):
        return {
            "required_obligations": [
                {"name": "roi_positive", "severity": "HARD", ...}
            ]
        }

# Mayor logic unchanged - still just checks attestations
```

```python
# WRONG: Add new decision logic
class MayorV3:
    def decide(self, briefcase, attestations):
        if business_vibes_good:  # ❌ NO! This breaks constitution
            return "SHIP"
```

**Principle:** Intelligence lives in **obligations** (what to check), not in **decision logic** (how to decide).

---

### 4. Mode A vs Mode B is Claim Routing, Not Special Logic
```python
if claim_type == "COMMENTARY":
    obligations = []  # Fast path - no verification needed
elif claim_type == "CHANGE_REQUEST":
    obligations = [...]  # Slow path - requires receipts
```

Mayor doesn't know about Mode A vs Mode B. It only knows:
```python
if len(O_HARD) == 0:
    return "SHIP"  # Mode A emerges naturally
else:
    return "SHIP" if all_satisfied else "NO_SHIP"  # Mode B
```

This keeps the kernel **simple and deterministic**.

---

### 5. EV District is the "Kernel Immune System"
EV doesn't just verify code - it verifies the **governance process itself**:

```
EV Obligations:
1. attestation_ledger_written     → "Did Factory do its job?"
2. mayor_emits_single_decision    → "Did Mayor stay sovereign?"
3. replay_determinism_hash_match  → "Is the kernel deterministic?"
4. briefcase_schema_valid         → "Did telemetry leak in?"
```

If EV fails → **system is unsafe**, regardless of whether code tests pass.

This creates a **self-correcting architecture** where the kernel checks itself.
