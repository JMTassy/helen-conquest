## Constitutional Guarantees

| Guarantee | How Enforced | Location |
|-----------|--------------|----------|
| **Bot does NOT write to main** | All changes through PR-only model | `scripts/bibliotheque_manifest.py` (generates but doesn't merge) |
| **Receipts are mandatory** | Manifest generation required before PR | `receipts/bibliotheque_intake_manifest.json` |
| **No self-attestation** | Bot proposes, CI validates, tribunal merges | Workflow: inbox → manifest → PR → CI → human decision |
| **Content never mutated** | Manifest hashes raw bytes as-is | `collect_inbox_items()` reads bytes, no interpretation |
| **Determinism verified** | Same file regenerated → same hash | CI determinism check (can be run locally too) |
| **Full audit trail** | Git history + manifest records | Every commit links to manifest hash in message |
| **Replayable** | Canonical JSON, deterministic normalization | `verify_intake_normalization.py` script |

## Constitutional Affirmation

This system enforces:

✅ **K0: Authority Separation**
Only registered attestors can sign decisions. Key registry immutable.

✅ **K1: Fail-Closed**
Missing evidence → NO_SHIP (safe default). No soft confidence levels.

✅ **K3: Quorum-by-Class**
Decisions require attestations from distinct agent classes. Single-source decisions rejected.

✅ **K5: Determinism**
Same claim + same policy → identical decision. Verified by 200-iteration replay.

✅ **K7: Policy Pinning**
Policy hash recorded immutably. Old decisions remain valid under old policy versions.

✅ **No Autonomous Writes to Main**
All changes through PR. Bot proposes, CI validates, tribunal merges.

✅ **Receipts Mandatory**
Every intake item hashed. Manifest proves exactly what was offered, when, with what hash.

✅ **No Self-Attestation**
Bot never signs its own outputs. Separation of proposer, validator, and decision-maker.
