## What This Is

**Street 1 with a conscious ledger** = the system emits an append-only event trace of its own operation, which can be replayed and verified.

**Not metaphysics.** Pure audit trail:
- Every interaction (user message, NPC reply, memory extraction) → logged event
- Same seed → identical event sequence → identical receipt hash
- Receipt hash proves: "this run happened, deterministically"

**Core principle**: NO RECEIPT = NO CLAIM.

## Architecture: 3 Layers

### L0: Event Ledger (Source of Truth)

**File**: `runs/street1/events.ndjson` (append-only)

Schema: `street1_event.v1.schema.json`

Each line is a JSON object with:
- `schema_version`: "STREET1_EVENT_V1"
- `run_id`: deterministic run identifier (seed-based)
- `t`: tick counter (world.tick at time of event)
- `type`: event category (OBS, BND, CHK, END, etc.)
- `prev_hash`, `hash`: sha256 chain for immutability

**Event Types**:
- `OBS`: Observation (user message, NPC reply, world delta, autonomous trigger)
- `BND`: Bound constraint (memory extraction result)
- `CHK`: Check/linter gate (determinism proof)
- `END`: Termination with receipt_sha

**Sample**:
```json
{"schema_version":"STREET1_EVENT_V1","run_id":"STREET1-42-1771696011577","t":0,"type":"OBS","sub_type":"session_start","actor":"SYSTEM","detail":"RESET_RUN with seed=42","prev_hash":null,"hash":"ea775fd6ddfdf4f9"}
{"schema_version":"STREET1_EVENT_V1","run_id":"STREET1-42-1771696011577","t":1,"type":"BND","sub_type":"memory_extraction","actor":"LNSA","facts":["Timeline: 2 weeks"],"prev_hash":"...","hash":"..."}
{"schema_version":"STREET1_EVENT_V1","run_id":"STREET1-42-1771696011577","t":1,"type":"END","sub_type":"session_end","actor":"HELEN","outcome":"DELIVER","reason":"normal","prev_hash":"...","hash":"efcc1843353dd80d"}
```

### L1: Extracted Facts (Recomputable)

**Not yet materialized**, but the concept:
- Parse L0 events
- Extract `BND` events (memory_extraction)
- Collect all `facts` arrays
- Store in `helen_memory.json`

This layer is **proof that memory is additive** (S3: append-only, never erased).

### L2: Summary Snapshot (Derived)

**File**: `runs/street1/summary.json`

Schema: `street1_summary.v1.schema.json`

Generated by `scripts/street1_rollup.py`:
- Scans all events
- Extracts facts, NPC replies, NPC-NPC interactions, final positions
- Includes `receipt_sha` (proof that summary is anchored to event stream)

**Sample**:
```json
{
  "schema_version": "STREET1_SUMMARY_V1",
  "run_id": "STREET1-42-1771696011577",
  "seed": 42,
  "event_count": 8,
  "facts_extracted": [
    {"fact": "Timeline: 2 weeks", "t": 1, "npc_id": null}
  ],
  "npc_replies": [
    {"npc_id": "olivia", "t": 1, "text": "I see you mentioned...", "source": "fallback"}
  ],
  "receipt_sha": "efcc1843353dd80d"
}
```

## Design Decisions

### Why Append-Only, Not Mutable DB?

**NDJSON** (newline-delimited JSON):
- ✅ No external dependencies (Node.js + Python built-ins only)
- ✅ Immutable by design (add only, never edit)
- ✅ Human-readable (one JSON object per line)
- ✅ Replayable (can read line-by-line in any language)
- ❌ Not queryable (but that's OK; use rollups for analysis)

### Why Hash-Chain Lightweight?

**Per-line sha256 hash**:
- ✅ Proves "nothing was inserted before this line"
- ✅ Simple: `hash(prev_hash + line_json)`
- ✅ No Merkle tree overhead
- ❌ Not a blockchain (no distributed consensus)

### Why Receipt as Final Event?

**`END` event with `receipt_sha`**:
- ✅ Proof that session terminated normally
- ✅ Receipt = hash of entire file (proves no post-facto edits)
- ✅ Can be validated: `sha256 events.ndjson == summary.receipt_sha`
